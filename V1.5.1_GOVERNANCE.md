# v1.5.1 Alpha 治理与数据完整性热修复

## 结论

v1.5.1 撤销 `quality_momentum_v1_5` 对严格通道默认配置的替换。生产默认 Alpha 恢复为 `legacy_v1_4`；v1.5 公式和权重保持冻结，继续作为可显式运行的实验候选，但其默认晋级决定记录为 `rejected`。

这不是删除研究结果，也不是根据历史结果再次调权。修复的目标是让代码默认值、验证结论和报告身份一致，避免把尚未通过实证验收的候选误当成成熟策略。

## 决策依据

绑定同一公开数据缓存的本地历史回放显示，`quality_momentum_v1_5` 相对同约束公开研究基准 `momentum_focus_15` 的表现并不一致：

| 区间 | momentum_focus_15 年化 | quality_momentum_v1_5 年化 | 年化改善 | 最大回撤改善（正值更好） | 夏普差 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 2013–2017 开发期 | 17.16% | 19.14% | +1.97 pp | -0.71 pp | +0.06 |
| 2018–2021 验证期 | 16.23% | 12.58% | -3.65 pp | -2.43 pp | -0.13 |
| 2022–2025 已查看历史保留期 | 5.18% | 5.22% | +0.05 pp | +7.09 pp | +0.03 |
| 2013–2025 全期 | 13.06% | 12.73% | -0.33 pp | -0.71 pp | -0.01 |

该对照可以研究选股 Alpha 的边际变化，但 `momentum_focus_15` 本身是公开研究方案，不等同于严格通道的 `legacy_v1_4` 默认配置。因此，即使候选在该表中全面胜出，也仍需严格通道和冻结后的前瞻数据才能完成默认晋级。

逐因子删除还显示，删除 `low_downside_vol` 或 `drawdown_quality` 后部分区间反而改善。由于删除因子会使剩余权重重新归一化，且因子彼此相关，这只能视为当前组合下缺乏稳定边际证据，不能解释为单因子的独立因果结论，也不能据此继续使用已查看数据调权。

## 代码治理

- `StrategyConfig()` 默认使用 `legacy_v1_4` 权重。
- YAML 新增逻辑配置项 `strategy.alpha_profile`：支持 `legacy_v1_4`、`quality_momentum_v1_5` 和 `custom`。
- 命名配置与显式权重冲突时直接报错；自定义权重必须声明 `custom`。
- v1.5.0 生成的旧 YAML 即使没有 `alpha_profile`，也会根据完整权重识别为实验候选，避免升级后静默改变其含义。
- `resolved_config.yaml`、`metrics.json`、HTML 报告和研究清单记录实际识别出的配置及其生命周期状态、默认晋级决定和原因。
- `quality_momentum_v1_5` 的生命周期为 `experimental`，默认晋级决定为 `rejected`；自定义权重为 `custom_unreviewed`。
- 严格研究套件先把输出目录解析为绝对路径，修复直接调用 API 且传入相对目录时结果封存重复拼接路径的问题。
- 公开研究遇到某方案完全没有调仓记录时，年化单边换手明确记为 0，不再因空表缺少 `execution_date` 而中断整批报告。

显式复核 v1.5 候选：

```yaml
strategy:
  alpha_profile: quality_momentum_v1_5
```

自定义权重：

```yaml
strategy:
  alpha_profile: custom
  momentum_12_1_weight: 0.35
  momentum_6_1_weight: 0.20
  fip_momentum_weight: 0.00
  trend_weight: 0.15
  low_volatility_weight: 0.20
  low_downside_volatility_weight: 0.00
  drawdown_quality_weight: 0.00
  liquidity_weight: 0.10
```

## 公开数据完整性

缓存 SHA256 校验只能证明已有文件没有变化，不能证明历史股票池没有缺口。v1.5.1 因此在每个 `data_quality.json` 中新增：

- `members_without_any_bars`：相关历史成员中完全没有行情的证券；
- `month_end_quote_coverage`：逐个信号月的成分数、可用行情数、覆盖率和缺失证券列表；
- `month_end_quote_coverage_below_threshold`：低于默认 95% 告警阈值的完整月份记录；
- `missing_execution_event_count` 与 `missing_execution_symbol_list`：执行日行情缺失的事件数和去重证券；
- `data_quality_status` 与 `data_quality_warnings`：机器可读的通过/告警状态。

覆盖率不足目前产生显式告警而不自动终止回测，因为停牌也可能造成月末缺报。研究者必须检查详细列表并判断是正常停牌、成分区间错误还是下载缺失；报告不再把最低覆盖率未经验证地归因于“集中停牌”。

## 后续边界

v1.6 的组合与成交增强应以 `legacy_v1_4` 为生产基线，同时把冻结的 v1.5 候选作为独立实验臂。组合层改动必须在相同 Alpha、相同数据指纹和相同交易成本口径下比较，避免把选股变化与组合/成交变化混在一起。
