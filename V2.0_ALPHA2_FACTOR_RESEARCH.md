# V2.0 Alpha 2：PIT 基本面与估值因子研究层

版本：`2.0.0a4`

## 1. 阶段目标与边界

Alpha 1 解决了“历史某日当时能看到什么财报和估值数据”。Alpha 2 在同一不可变 PIT 契约上建立研究层，回答：

- 每个候选因子在逐月沪深 300 历史成分中的实际覆盖率是多少；
- 因子与未来 21/63 个交易日收益的截面秩相关是否稳定；
- 收益是否随因子分组近似单调，而不是由少量离群点制造；
- 因子是否主要复制行业或市值暴露；
- 删除任一因子、提高成本和向前滚动后证据是否仍存在。

本阶段不把 PIT 因子接入 `MultiFactorStrategy`，不改变 `legacy_v1_4 + inverse_vol_v1_4 + fixed_bps` 生产默认，也不自动寻优权重。即使历史指标全部越过阈值，程序最多给出 `eligible_for_manual_candidate_review`，不能自动晋级生产。

## 2. 固定因子注册表

所有方向统一为“数值越高越好”，截面内先缩尾再标准化。缺失值保持缺失，不用零或截面均值填充。

| 因子 | 家族 | 固定定义 |
| --- | --- | --- |
| `roe_quality` | 盈利 | 最新可见 `roe_pct` |
| `roa_quality` | 盈利 | 最新可见 `roa_pct` |
| `gross_margin_quality` | 盈利 | 最新可见 `gross_margin_pct` |
| `cashflow_quality` | 质量 | `operating_cash_to_revenue_pct` |
| `low_leverage_quality` | 质量 | `-debt_to_assets_pct` |
| `revenue_growth` | 成长 | `revenue_yoy_pct` |
| `earnings_growth` | 成长 | `net_income_yoy_pct` |
| `earnings_yield` | 估值 | `1 / pe_ttm`，仅保留正 PE |
| `book_to_price` | 估值 | `1 / pb`，仅保留正 PB |
| `sales_yield` | 估值 | `1 / ps_ttm`，仅保留正 PS |
| `dividend_yield` | 估值 | `dividend_yield_ttm_pct` |

研究候选 `fundamental_value_composite_v2_alpha2` 是可用因子截面 z-score 的固定等权均值。默认每只股票至少有 4 个因子才形成组合分数。它没有经过拟合，生产权重固定为 0。

本阶段刻意不从原始累计利润/现金流自行拼接单季度或 TTM 数据，避免把报表累计口径误当成单季度口径。银行、券商等行业的杠杆和利润率仍可能口径不同，因此行业暴露是必查项。

## 3. 时序与收益标签

1. 信号日采用回测区间内每月最后一个交易日。
2. 股票池使用信号日之前最近一次历史指数成分快照。
3. 财报只使用 `available_date <= signal_date` 的最新可见修订，并遵守最大财报年龄。
4. 估值只使用当时可见的最近日终记录，并遵守最大估值年龄。
5. 行业使用信号日有效的历史行业区间，市值使用 PIT `daily_basic` 市值。
6. 未来收益使用复权收盘价，只作为研究标签；它不会参与因子计算。
7. 停牌价格最多按配置中的陈旧价格天数向前填充，超限或退市后标签缺失。

测试会修改未来公告、未来修订和未来估值，要求历史因子面板逐字相同。

## 4. 研究命令

先准备并验证基础行情和 PIT 侧车：

```bash
python run.py validate-data --config config.yaml
python run.py pit-verify --config config.yaml
```

运行默认 Alpha 2 研究：

```bash
python run.py pit-research \
  --config config.yaml \
  --output results/pit_factor_research_v2_alpha2 \
  --factors all \
  --horizons 21,63 \
  --quantiles 5 \
  --minimum-factors-per-symbol 4 \
  --minimum-ic-observations 20 \
  --cost-bps 5,10,20 \
  --train-years 5 \
  --test-years 1
```

输出目录必须为空。程序拒绝覆盖或混入旧产物，避免旧结果被新 manifest 误封存。

## 5. 输出与复现身份

| 文件 | 内容 |
| --- | --- |
| `pit_factor_registry.csv` | 因子公式、输入字段、方向和生命周期 |
| `pit_factor_panel.csv.gz` | 信号日、证券、PIT 原始值/z-score、财报期/可见日/修订哈希、行业、市值和未来收益标签 |
| `pit_factor_coverage.csv` | 逐信号日、逐因子覆盖率 |
| `pit_factor_ic.csv` / `pit_factor_ic_summary.csv` | 逐期 Spearman IC 与汇总 IC/ICIR/正 IC 比例 |
| `pit_factor_quantile_returns.csv` / `pit_factor_quantile_summary.csv` | 分组收益和最高组减最低组收益 |
| `pit_factor_exposures.csv` / `pit_factor_exposure_summary.csv` | 市值秩相关、行业解释度和行业均值偏离 |
| `pit_factor_ablation.csv` | 完整等权组合与逐因子删除结果 |
| `pit_factor_rolling.csv` | 固定定义、无拟合的扩展研究窗/非重叠测试窗证据 |
| `pit_factor_cost_stress.csv` / `pit_factor_cost_summary.csv` | 顶部分组等权组合按相邻月末非重叠收益计算的换手和 5/10/20 bps 权重级成本压力 |
| `pit_factor_governance.json` | 固定阈值、逐项检查、生命周期和晋级决定 |
| `reproducibility.json` | 配置、源码、依赖、Git、基础行情及 PIT manifest 身份 |
| `experiment_registry.jsonl` | 研究协议与实验指纹 |
| `artifact_manifest.json` | 全部产物的大小、SHA256 和集合指纹 |

运行指纹同时包含基础行情 manifest 和 PIT manifest。研究 manifest 另显式保存两份数据指纹，防止只绑定行情却遗漏财报数据。

## 6. 治理阈值

组合因子默认以最短前瞻期作为主要口径，至少检查：24 个信号日、覆盖率、平均 IC、正 IC 比例、最高/最低分组差、滚动窗口一致性、行业/市值暴露和最高成本压力下的净结果。

成本组合只按信号分数选股，不会先看未来收益是否存在再决定样本；未来标签缺失单独输出 `outcome_coverage`，治理要求最高成本情景的最低结果覆盖率不低于 90%。

这些阈值是“是否值得人工复核”的过滤器，不是统计显著性的充分证明，更不是 15% 年化收益承诺。2013–2025 历史区间已被反复查看，`untouched_holdout_certified` 固定为 `false`。真正晋级仍需要：

1. 绑定真实 Tushare PIT 指纹完成全量研究；
2. 检查缺失模式是否集中在特定行业/年代；
3. 冻结候选定义后在未参与修改的新时期或模拟盘前瞻验证；
4. 若进入策略候选，再单独做严格账本回测和价格 Alpha 增量归因。

## 7. 已知限制

- 当前成长/盈利因子沿用 Tushare 财务指标口径，尚未按银行、保险、券商建立专用模型。
- 21/63 日标签可能重叠，因此 IC 和分组结果不能当作独立同分布样本。
- 成本压力是月度等权目标的权重级估算，不模拟涨跌停排队、整手、最低佣金和容量重试。
- 本阶段没有云端 Tushare Token，合成测试只证明时序、公式和产物治理正确，不提供真实 Alpha 证据。
