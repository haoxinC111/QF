# v1.6 组合与成交增强协议

## 结论边界

v1.6 在不改变 Alpha 公式、截面排名和入选证券集合的前提下，新增两项可显式启用的实验能力：

- `shrinkage_min_variance_v1_6`：收缩协方差、最小方差混合与当前持仓平滑；
- `square_root_v1_6`：基于信号时点波动率和滞后 ADV20 参与率的平方根市场冲击。

两项模型均为 `experimental/pending_validation`。生产默认继续使用 `inverse_vol_v1_4 + fixed_bps`，因此升级代码不会静默改变旧回测结果。v1.6 的目标是提高组合风险表达和成交假设的保守性，不承诺 15% 年化收益，也不把更高交易成本导致的收益下降解释为策略退化。

## 组合模型

### 冻结参数

| 参数 | v1.6 值 | 含义 |
| --- | ---: | --- |
| `covariance_lookback_days` | 120 | 信号日及以前的总回报日收益窗口 |
| `minimum_covariance_observations` | 60 | 任一入选证券的最低有效观测数 |
| `covariance_shrinkage` | 0.50 | 样本协方差向对角阵收缩比例 |
| `minimum_variance_blend` | 0.50 | 最小方差偏好在风险目标中的占比 |
| `turnover_smoothing` | 0.50 | 当前实际持仓在最终偏好中的占比 |
| `covariance_ridge` | 0.000001 | 特征值正定化下限相对尺度 |

### 算法

对月末入选的同一组股票，以截至信号日可见的后复权总回报日收益估计协方差：

\[
\Sigma_s=(1-\lambda)\Sigma+\lambda\,diag(\Sigma),\quad \lambda=0.5
\]

两两有效观测不足时显式回退 `inverse_vol_v1_4`，并在 `portfolio_status` 写入 `fallback_insufficient_covariance`。两两协方差可能产生轻微非正定矩阵，因此程序对特征值设置与方差尺度相关的正下限，再计算广义逆。最小方差偏好为：

\[
p_{mv}=normalize\left(max\left(\Sigma_s^+\mathbf{1},0\right)\right)
\]

它与旧版逆波动率偏好各占 50%，形成未平滑风险目标。若已有实际持仓，再把信号日收盘实际权重与风险目标各占 50% 混合。每一步最终都重新施加总敞口、单股上限和行业上限；组合不会为了凑满仓突破硬约束。

这不是 Barra 风险模型，也不估计预期收益。负的无约束最小方差权重会被截为零，因此它是可解释、无额外求解器依赖的长-only 近似，不应冒充精确二次规划解。

### 审计字段

`selections.csv` 新增：

- `portfolio_model` / `portfolio_status`；
- `covariance_observations`；
- `raw_target_weight`：换手平滑前的风险目标；
- `current_weight`：信号日收盘实际持仓权重；
- `target_weight_change`：最终目标减当前权重。

## 成交模型

### 平方根冲击

订单先受信号日已知的 20 日平均成交额限制。对计划实际成交的参考开盘金额，参与率为 `q / ADV20`。年化信号波动率低于 10% 时按 10% 计算，市场冲击为：

\[
Impact_{bps}=min\left(50,10000\times0.5\times
\frac{\max(\sigma_{ann},10\%)}{\sqrt{252}}\times
\sqrt{\frac{q}{ADV20}}\right)
\]

最终模型滑点等于固定 5 bps 加市场冲击，买入向上、卖出向下，并继续受当日涨跌停价限制。容量和冲击只使用信号日已经知道的 ADV20 与波动率；程序不会用执行日全天成交额决定开盘成交量，因为那会引入日内未来信息。

### 部分成交与订单生命周期

容量不足时按整手部分成交，剩余目标保持冻结并在最多 3 个交易日内重试。v1.6 修复了“买单容量不足一个整手却被误记为 `below_lot` 并立即结束”的问题：现在会记为 `participation_cap`，进入重试并最终明确成交或取消。停牌、涨跌停、T+1、执行日 ST、现金不足和新调仓覆盖仍分别记录原因。

`orders.csv` / `trades.csv` 新增请求股数、剩余股数、成交比例、参考开盘价、滞后 ADV20、容量上限、参与率、信号波动率、固定滑点、市场冲击、模型/实际滑点和估算成本。报告同时输出平均/最大参与率、部分成交笔数及估算冲击成本。

日线模型仍不能知道集合竞价真实排队、盘口深度和券商实际回报；这里的“部分成交”是容量模型结果，不是历史逐笔成交重放。

## 四臂归因协议

严格研究命令新增 `implementation` 模式：

```bash
python run.py research --config config.yaml --modes implementation
```

`implementation_comparison.csv` 固定同一 Alpha、数据指纹和其他参数，只运行：

1. `baseline_v1_5_1`：旧组合 + 固定滑点；
2. `portfolio_only_v1_6`：新组合 + 固定滑点；
3. `execution_only_v1_6`：旧组合 + 平方根冲击；
4. `combined_v1_6`：新组合 + 平方根冲击。

只有第一与第二臂可用于归因组合构建；第一与第三臂用于量化更保守成交假设的成本。不得拿第一臂 Alpha 与第四臂的总差异全部归因于组合模型。

现有 2012–2025 公开缓存可直接运行权重级近似四臂：

```bash
python run.py public-implementation \
  --membership ../public_data/index-constitution/history/csi300.csv \
  --cache ../public_data/sina_csi300 \
  --output results/public_implementation_v1_6 \
  --initial-capital 1000000
```

公开通道的平方根冲击按 100 万元初始资金估算，但不模拟整手、最低佣金、涨跌停排队、容量部分成交与重试；产物会写明 `strict_execution_equivalent=false`。它适合复用已有数据判断方向，不替代严格通道。

显式运行组合与成交实验模型：

```yaml
portfolio:
  construction_model: shrinkage_min_variance_v1_6

execution:
  market_impact_model: square_root_v1_6
```

## 晋级条件

真实缓存验证必须绑定相同数据 SHA256、源码提交和配置指纹。组合模型至少应满足：滚动窗口多数风险调整指标不差、全期实际换手下降、最大回撤不出现不可解释恶化、收益变化没有依赖单一区间。成交模型的晋级依据是无未来函数、压力稳定和与模拟盘/券商成交的校准，而不是“提高收益”。2013–2025 已被反复查看，任何该区间结果都只能叫历史验证；真正前瞻证据应从未参与改参的新数据或模拟盘开始。
