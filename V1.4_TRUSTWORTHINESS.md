# v1.4 结果可信度改造说明

## v1.4.1 热修复

v1.4.1 不改变策略、交易模型或历史净值计算，只修复发布与审计链路：

1. `results/` 不再作为源码的一部分被 Git 追踪。
2. 所有回测和研究输出生成 `artifact_manifest.json`，记录逐文件大小、SHA256 和集合指纹。
3. `experiment_registry.jsonl` 从文件名列表升级为带路径、大小和 SHA256 的产物清单。
4. 新增 `result-verify` 命令，可检测结果缺失、被修改以及严格模式下的未封存文件。
5. 复现清单同时记录 `akracer`、`mini-racer`、`py-mini-racer` 及实际加载的 `py_mini_racer` 模块来源和文件 SHA256。
6. uv 锁定安装显式使用 `--default-index https://pypi.org/simple`，不再建议改写锁文件。

## 改造范围

v1.4 不增加新 Alpha 因子，目标是让同一份代码、配置和数据产生可核验的同口径结果。

1. 公开通道遇到执行日停牌时，旧持仓市值保持锁定。若锁仓与新目标导致资金不足，只缩放可执行目标，并根据实际目标重新计算买卖额、费用和现金。
2. 严格通道新增独立 `regime` 数据序列。默认 `000300.SH` 价格指数只决定风险开关，`H00300.CSI` 全收益指数只用于业绩比较。
3. 严格缓存升级为 v4，所有回测实际读取的文件都记录大小和 SHA256；读取缓存时先做完整性校验。
4. 公开缓存保存行情文件与历史成分文件的联合指纹。v1.3 缓存可以原地封存，不需要重新下载。
5. 回测和研究输出新增 `reproducibility.json`，记录配置、源码、Git commit/脏状态、数据 manifest、Python 与依赖版本，以及组合运行指纹。
6. 提交 `uv.lock`，使依赖解析可锁定到确定版本。
7. 输出 `experiment_registry.jsonl`，按运行指纹登记实验协议，并把 2022–2025 明确标为已查看历史保留期。

## 结果口径变化

公开通道历史结果必须重新运行。只有在“存在停牌旧持仓，同时新目标使资金不足”的调仓日，净值和换手才会发生变化；新结果更保守，也更符合不可交易约束。严格通道风险开关从全收益基准切换到独立价格指数后，风险状态日期可能变化，因此 v1.3 与 v1.4 的严格结果不能直接当作同策略复跑比较。

## 升级步骤

公开缓存：

```bash
python run.py public-verify --membership /path/to/csi300.csv \
  --cache /path/to/public_cache --seal-legacy
python run.py public-verify --membership /path/to/csi300.csv \
  --cache /path/to/public_cache
```

严格缓存：在 `config.yaml` 增加 `data.regime_index: 000300.SH`，把 `data.refresh` 暂时设为 `true` 后重新运行 `download`。v4 下载完成后改回 `false`。

精确环境：

```bash
uv sync --locked
uv sync --locked --extra public --default-index https://pypi.org/simple
```

## 验收标准

- 停牌锁定持仓在资金缩放前后市值完全相同。
- 调仓后满足 `现金 + 持仓 + 当日费用 = 调仓前净值`。
- 改变业绩基准走势不会改变风险开关；改变择时指数走势会改变风险开关。
- 任一已封存输入文件发生变化，校验立即失败。
- 每次回测和研究均生成可追溯到配置、源码、依赖和数据的运行指纹。
- 完整离线测试与 demo 成功，不依赖 Tushare Token。
