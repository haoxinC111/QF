# V2.0 Alpha 2 验收记录

版本：`2.0.0a5`

## 验收目标

本阶段验收 PIT 基本面/估值研究管线的工程正确性，不验收收益目标，也不把研究候选晋级为生产策略。

必须满足：

- 固定因子注册表和等权候选的生产权重均为 0；
- 月末信号、动态成分、历史行业和 PIT 可见性严格对齐；
- 未来公告、修订和估值扰动不能改变过去面板；
- 覆盖、IC、分组、行业/市值暴露、消融、成本和滚动结果可独立审计；
- 基础行情及 PIT 两份数据指纹同时进入研究身份；
- 非空结果目录拒绝覆盖，严格产物 SHA256 校验通过；
- v1.6 离线 demo 冻结结果不变。

## 建议验证命令

```bash
env -u UV_DEFAULT_INDEX uv lock --check --default-index https://pypi.org/simple
env -u UV_DEFAULT_INDEX uv sync --locked --extra public --default-index https://pypi.org/simple
uv run --no-sync ruff check src tests
uv run --no-sync python -W error::FutureWarning -m unittest discover -s tests -v
uv run --no-sync python run.py demo --seed 7 --output results/demo_v2_alpha2
uv run --no-sync python run.py result-verify --output results/demo_v2_alpha2 --strict
```

真实 PIT 验收需另执行：

```bash
uv run --no-sync python run.py pit-verify --config config.yaml
uv run --no-sync python run.py pit-research \
  --config config.yaml \
  --output results/pit_factor_research_v2_alpha2
uv run --no-sync python run.py result-verify \
  --output results/pit_factor_research_v2_alpha2 --strict
```

## Hotfix：`v2.0.0a4 → v2.0.0a5`

`v2.0.0a4` 发布后本地复现发现两个问题：

1. `ruff` 未写入 `pyproject.toml`/`uv.lock`，`uv run --no-sync ruff check src tests` 在干净环境找不到命令。
2. CI 只测 Python 3.12，但项目声明支持 `>=3.10`。

`v2.0.0a5` 修复：

- 在 `[dependency-groups] dev` 中锁定 `ruff==0.15.21`，并重新生成 PyPI 版 `uv.lock`；
- 更新 `.github/workflows/ci.yml`，增加 Python 3.10 + 3.12 矩阵，并在测试前运行 `ruff check src tests`；
- 版本号统一为 `2.0.0a5`。

功能代码、PIT 研究实现和 118 项测试结果与 `v2.0.0a4` 完全一致。
