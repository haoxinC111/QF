## 4. 官方接口契约复核

实现按 Tushare 官方文档使用：

- `income`、`balancesheet`、`cashflow` 按单只证券获取历史数据，显式请求 `report_type=1`；
- 三张报表保留公告日、实际公告日、公司类型和更新标识；
- `fina_indicator` 不请求不存在的 `f_ann_date/report_type/comp_type`，但保留官方提供的 `update_flag`；
- `daily_basic` 的总/流通市值以万元人民币存储并在规范字段名中明确单位。

## 5. 未在本环境完成的真实数据验收

本环境没有用户的 Tushare Token，也没有使用用户账户消耗接口配额，因此没有执行真实沪深 300 全量 `pit-download`。这不是单元测试可以替代的环节。具备权限的本地环境应完成：

```bash

python run.py pit-download --config config.yaml
python run.py pit-verify --config config.yaml
python run.py pit-snapshot --config config.yaml --date 2025-12-31 \
  --output results/pit_snapshot_2025-12-31.csv
```

验收时需要记录财报/估值行数、证券覆盖率、PIT 数据指纹、基础行情指纹、缺失证券和下载日期。由于上游历史会修订，不同日期下载的数据只能称为新的可审计快照，不能自动称为字节级严格复现。

## 7. Hotfix：`uv.lock` 镜像源修复（`v2.0.0a2`）

`v2.0.0a1` 发布后发现 `uv.lock` 被本地 `UV_DEFAULT_INDEX=https://mirrors.aliyun.com/pypi/simple` 污染：lockfile revision 从 3 降到 2，所有包下载地址被改为阿里云镜像。按 README 使用官方 PyPI 时 `uv lock --check --default-index https://pypi.org/simple` 失败，影响跨环境严格复现。

`v2.0.0a2` 仅做以下修复：

- 在取消 `UV_DEFAULT_INDEX` 的环境下用 PyPI 重新生成 `uv.lock`，恢复 `revision = 3` 与官方下载地址；
- 更新 `CLAUDE.md`，说明 `uv run` 默认会隐式同步，也可能在阿里云环境下改写 `uv.lock`，应在同步/运行时同样取消 `UV_DEFAULT_INDEX` 或使用 `uv run --no-sync`；
- 功能代码、测试和 macOS 路径修复均保持不变。

复现命令（必须在无 `UV_DEFAULT_INDEX` 的环境中执行）：

```bash
uv lock --check --default-index https://pypi.org/simple
uv sync --locked --extra public --default-index https://pypi.org/simple
ruff check src tests
python -W error::FutureWarning -m unittest discover -s tests
```

以上检查在 `v2.0.0a2` 上全部通过。

## 8. 版本号对齐修复（`v2.0.0a3`）

`v2.0.0a2` 发布后发现 `pyproject.toml`、`src/ashare_quant/__init__.py` 和 `uv.lock` 中的项目版本仍停留在 `2.0.0a1`，与 Git 标签不一致。`v2.0.0a3` 统一将版本号修改为 `2.0.0a3`，并重新生成 PyPI 版 `uv.lock`。功能代码、测试与 v2.0.0a2 保持一致。
