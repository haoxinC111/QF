# V2.0 Alpha 3：PIT 候选影子接入与严格账本归因

版本：`2.0.0a7`

## 1. 阶段目标与边界

Alpha 2 用 PIT 财报/估值数据回答“因子在截面研究中是否有证据”。Alpha 3 继续回答另一个问题：固定 PIT 候选进入现有选股、组合和成交约束后，是否相对价格 Alpha 提供可成交的增量，而不是只在权重级分组收益中看起来有效。

本阶段仍不改变生产默认。`legacy_v1_4 + inverse_vol_v1_4 + fixed_bps` 保持原路径；PIT 候选无法写入 `AppConfig.strategy`，只能通过 `pit-shadow` 研究命令显式注入。候选生产权重固定为 0，即使全部历史门槛通过也最多进入前瞻模拟观察。

## 2. 固定候选与四臂归因

候选 `price_fundamental_hybrid_v2_alpha3` 固定使用 Alpha 2 的 11 个因子、每只股票至少 4 个有效因子，以及 25% PIT/75% 价格分数。25% 是预先冻结的研究权重，命令行不提供寻优入口。

四个臂共享相同的历史成分、ST/价格/趋势/流动性过滤、排名缓冲、组合模型、风险开关和严格成交引擎：

| 臂 | 排名分数 | 目的 |
| --- | --- | --- |
| `production_baseline` | 原生产价格 Alpha | 冻结生产基线 |
| `coverage_matched_price` | 只在 PIT 可覆盖股票中使用价格 Alpha | 单独测量 PIT 缺失筛选效应 |
| `pit_only_shadow` | PIT 等权组合 | 测量独立 PIT 排名效果 |
| `hybrid_fixed_25pct_pit` | 75% 价格 + 25% PIT | 固定混合候选 |

比较混合臂时主要以 `coverage_matched_price` 为控制，避免把“有财报数据的股票本身不同”错误归因给 PIT 分数。

## 3. 防前视结构

- 影子分数按每个可执行月末重新从 PIT 快照计算；财报和估值可见日不得晚于信号日。
- `PITShadowScorePanel` 明确拒绝 `forward_return_*`、`outcome_date_*` 和 `next_signal_date` 字段。
- Alpha 2 的未来收益标签不会传入 `MultiFactorStrategy` 或 `Backtester`。
- 未来公告、未来修订和未来估值扰动不能改变过去影子分数。
- 月末信号仍在下一交易日开盘执行，继续应用整手、T+1、停牌、涨跌停、容量、费用、部分成交和重试。

## 4. Alpha 2 前置门禁

`pit-shadow` 必须读取一个严格封存的 Alpha 2 研究目录，并逐项确认：

1. `artifact_manifest.json` 严格校验通过且没有未封存文件；
2. 使用完整固定因子注册表和每股至少 4 个因子；
3. Alpha 2 候选仍为 `fundamental_value_composite_v2_alpha2`；
4. 基础行情和 PIT 数据指纹与当前缓存完全一致；
5. Alpha 2 没有自动生产晋级，也没有修改生产默认。

Alpha 2 证据不足不会阻止生成诊断结果，但会把 Alpha 3 晋级决定锁为 `blocked_by_alpha2_evidence`。非 Tushare 或合成数据一律标记 `nonproduction_data_only`。

## 5. 使用方式

先完成真实 PIT 验收：

```bash
python run.py validate-data --config config.yaml
python run.py pit-verify --config config.yaml
python run.py pit-research \
  --config config.yaml \
  --output results/pit_factor_research_v2_alpha2
python run.py result-verify \
  --output results/pit_factor_research_v2_alpha2 --strict
```

再运行严格账本影子归因：

```bash
python run.py pit-shadow \
  --config config.yaml \
  --alpha2-research results/pit_factor_research_v2_alpha2 \
  --output results/pit_shadow_v2_alpha3 \
  --cost-bps 5,10,20
python run.py result-verify \
  --output results/pit_shadow_v2_alpha3 --strict
```

输出目录必须为空，防止旧结果混入新指纹。

## 6. 主要输出

| 文件 | 内容 |
| --- | --- |
| `pit_shadow_score_panel.csv.gz` | 不含未来收益的逐月 PIT 原始因子、z-score、组合分数与可见性摘要 |
| `pit_shadow_coverage.csv` | 逐月可评分股票数量与覆盖率 |
| `pit_shadow_comparison.csv` | 四臂全期收益、风险、换手、费用及相对两类基线的差值 |
| `pit_shadow_annual_metrics.csv` | 各年度收益、夏普、回撤、换手和费用 |
| `pit_shadow_selection_attribution.csv` | 各臂逐月选股重合度与入选 PIT 分数 |
| `pit_shadow_cost_stress.csv` | 混合臂在固定滑点压力下的严格账本结果 |
| `arms/<arm>/` | 每个臂的净值、选股、订单、成交、公司行动、指标和期末持仓 |
| `pit_shadow_governance.json` | Alpha 2 门禁、固定阈值、证据和晋级决定 |
| `reproducibility.json` | 源码、Git、依赖、基础行情/PIT/Alpha2 研究身份 |
| `experiment_registry.jsonl` | 固定协议与实验指纹 |
| `artifact_manifest.json` | 全部输出的大小、SHA256 和集合指纹 |

## 7. 治理解释

治理检查覆盖 PIT 覆盖率、相对覆盖匹配基线的 CAGR/夏普增量、回撤恶化、换手倍率、年度一致性和最高成本情景。通过后决定也只能是 `eligible_for_forward_paper_tracking`，不能是生产晋级。

2013–2025 已被反复查看，因此 `untouched_holdout_certified=false`。历史严格回测不能证明未来年化达到 15%；真正的下一道门槛是冻结代码和数据身份后，使用新增时期或模拟盘做前瞻跟踪。

## 8. 已知限制

- 当前仍是日线严格回测，不重放集合竞价队列和逐笔盘口。
- 固定混合权重没有真实数据优化，也不能因一次历史结果而修改。
- PIT 覆盖可能随行业和年份系统性变化，必须同时查看覆盖匹配价格臂。
- Alpha 2 研究门禁和 Alpha 3 严格账本门禁都通过，也不替代人工财务口径核查。
