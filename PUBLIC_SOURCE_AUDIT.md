# 公开数据源审计记录

审计日期：2026-07-12（America/Los_Angeles）

本记录解释 v1.3 公开数据回测为什么选择当前数据源，以及哪些看似可用的数据被排除。结论先行：历史成分采用 `unliftedq/index-constitution` v0.6.2，行情采用新浪财经公开 HTTPS 日线与后复权因子；另用一个公开 Tushare 样本仓库对日收益做跨源校验。

## 最终采用

### 历史沪深300成分

- 来源：https://github.com/unliftedq/index-constitution
- 固定版本：v0.6.2，提交 `16d9d69fc0bf7f0f5e9aace868e16e26f2ecb5c2`
- 文件：`history/csi300.csv`
- 内容：1,225 个纳入/剔除区间，949 个唯一代码；本次 2012–2025 下载涉及 749 个唯一历史成员。
- 时点规则：`opt-in <= date` 且 `opt-out > date`，剔除日为排他边界。
- 许可：MIT。
- 局限：这是第三方对中证指数官方公告的规范化整理，不是中证公司原始机器接口；项目固定 tag 和提交号以避免数据静默漂移。

### 日线与复权

- 来源：新浪财经公开 HTTPS 历史行情文件及 `hfq.js` 后复权因子。
- 区间：2012-01-01 至 2025-12-31；公开回测从 2013-01-01 开始，前一年作为 252 日动量预热。
- 字段：开、高、低、收、成交量、成交额；股票和 ETF 使用后复权总回报价格，沪深300价格指数仅用于 200 日均线风险状态。
- 下载控制：逐代码 gzip 缓存、临时文件原子替换、结构/OHLC 校验、指数基准强制成功、失败代码记录、指数退避重试与断点续传。
- 文件级覆盖为 714/715 个正式回测期历史成员。唯一无 2012 年后行情的是 `SH600357`，但第三方成分文件仍把其成员区间记录到 2014-07-10；程序不会为它生成目标，并把这项源数据冲突留在清单中。月末当日有报价的成分覆盖中位数为 99.33%，最低 90.00% 出现在 2015 年集中停牌月份。后者主要是当日不可交易状态，不是行情文件缺失，策略保守地不对无收盘价证券生成新目标。
- 局限：接口没有服务等级协议，可能限流或修订；因此缓存清单、获取时间和源版本必须与结果一起保存。

### 跨源校验

- 参考来源：https://github.com/shaocongWu/Multivariate_Stock_Time_Series_Dataset
- 固定提交：`9d9e4a9b02bf76ef6bde1a66b9c5447ed78bc962`
- 参考数据说明：作者声明原始数据来自 Tushare Pro，覆盖 800 只沪深300/中证500股票的 2019-01-02 至 2021-01-29 日线。
- 校验方式：分别在每个数据源内部计算复权收盘日收益，再按证券和交易日做内连接；不比较绝对复权价位。
- 结果：531 只股票、267,511 个日收益对；代码级相关系数中位数 0.99999994，5% 分位数 0.99999417，日收益绝对差中位数约 0.00000109。

## 审计后排除

### `yifishbossman/financial-analyst-data-full`

- 页面：https://huggingface.co/datasets/yifishbossman/financial-analyst-data-full
- 页面声明约 5,450 只股票并包含行情、估值和财务数据，但远端实际完整日线字段约只覆盖 1,600 只，且明显偏向上海市场。
- 对 949 个历史沪深300唯一成员，OHLCV、成交额、复权因子和市值全字段同时可用的仅 486 个；若强行回测会产生严重的交易所和幸存者选择偏差。

### `jobs-git/StockChina-Minute`

- 页面：https://huggingface.co/datasets/jobs-git/StockChina-Minute
- 页面声明 5,267 只股票、2005–2023 分钟线；实际仓库文件列表约 1,450 个 CSV，全部为深圳代码。
- 历史沪深300月度成员覆盖约 27%–36%，不能构建可比股票池。

### BaoStock

- 官网：http://www.baostock.com/
- API 本身支持指定日期的沪深300成分与日线，理论口径合适；但客户端使用 `public-api.baostock.com:10030` 原始 TCP，本运行环境无法建立该端口连接。
- 这是环境连通性问题，不代表 BaoStock 数据质量结论。

### Yahoo Finance / yfinance

- 沪深股票及沪深300测试请求均返回 HTTP 429 / `YFRateLimitError`，共享出口当前不可重复使用。

### `21hbguo/stocksim-v2`

- 页面：https://github.com/21hbguo/stocksim-v2
- 仓库内 SQLite 仅 390 只股票、365,065 条日线，主要从 2019 年开始；股票集合近似当前池，无法替代历史动态成分，容易产生幸存者偏差。

### 东方财富公开日线

- 单证券验证成功，字段完整；批量约 35 个证券后响应出现持续超时/空 JSON，说明当前出口触发上游限流。
- v1.3 保留 `--source eastmoney` 备用适配器，但本次正式结果使用独立新浪缓存，不混合两个行情源。

## 结论边界

公开通道解决了“无 Token 时是否能用真实历史价格和动态成分验证策略方向”的问题，但没有解决历史 ST、历史申万行业、历史市值、逐日涨跌停价、完整停牌和公司行动账本。因此公开通道采用权重级成本回测，不应把结果解释为券商可成交净收益。最终候选策略必须回到项目原有的严格 Tushare 通道复核。
