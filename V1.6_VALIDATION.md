# v1.6 工程验证记录

## 验收状态

v1.6.0 的组合与成交模块已经完成工程实现，生产默认保持 `inverse_vol_v1_4 + fixed_bps`。新模型 `shrinkage_min_variance_v1_6 + square_root_v1_6` 为 `experimental/pending_validation`，没有因合成数据结果自动晋级。

已完成的自动化验收：

- 依赖锁文件可通过 `uv lock --check`；
- 锁定 `public` 依赖环境中 83 项单元/集成测试全部通过；其中包含真实 MiniRacer 六 worker 共用单运行时与子进程压力测试；
- Ruff 静态检查通过，`git diff --check` 通过；
- 旧生产配置的确定性黄金指标保持不变；
- 用同一组合成公开缓存对比 `v1.5.1`（`0e66df6`）与 v1.6 默认路径，129 个交易日净值字节 SHA256 均为 `9f9b1d7f1a7a6883481e52cf60adc4f8d951b0aa56ec9a4e6a1311429bd893e9`，成本和换手也逐值相同；
- v1.6 协方差估计只读取信号日及以前收益，未来价格扰动不改变目标；
- 协方差观测不足时显式回退旧权重模型；
- 单股、行业和总敞口硬约束在新模型下继续成立；
- 平方根冲击随波动率/参与率单调增加，并受 50 bps 上限约束；
- 请求、部分成交、剩余订单、参与率、冲击和成本均写入订单账本；
- 容量部分成交会跨 3 个交易日重试；容量低于一个整手时不会被静默当作目标完成；
- 严格研究与公开权重级研究均包含固定 Alpha 四臂归因；
- 公开四臂输出通过严格产物 SHA256 校验。
- 独立 V8 压力测试以 6 workers、每轮 600 任务、3 轮完成 1,800 次解码，无崩溃。

## 离线演示

同一份 `seed=7`、2018–2025 合成行情结果：

| 模型 | 年化 | 最大回撤 | 夏普 | 年化单边换手 | 券商/法定费用 | 估算冲击成本 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| 旧组合 + 固定滑点 | 0.74% | -12.15% | 0.138 | 210.45% | 26,950.27 | 0.00 |
| v1.6 组合 + 平方根冲击 | 0.71% | -12.56% | 0.134 | 187.35% | 24,652.37 | 2,891.18 |

这组假行情仅证明链路能够运行并显示换手平滑生效：实验组合的年化单边换手下降约 23.10 个百分点。收益和回撤没有改善，不能把它宣传为 Alpha 增强；平方根冲击使成交假设更保守，其成本为正是预期行为。

另一组短周期合成四臂也验证了归因方向：仅切换平方根成交时估算冲击成本从 0 变为正；仅切换组合时换手发生变化；组合与成交没有被混在同一个不可拆分的方案中。

## 真实公开数据状态

仓库不包含用户本地的 2012–2025 行情缓存，因此本次云端环境没有生成绑定该缓存指纹的真实 v1.6 数值。代码已提供：

```bash
python run.py public-implementation \
  --membership ../public_data/index-constitution/history/csi300.csv \
  --cache ../public_data/sina_csi300 \
  --output results/public_implementation_v1_6
```

本地运行后应先执行：

```bash
python run.py result-verify \
  --output results/public_implementation_v1_6 --strict
```

公开结果必须与 `reproducibility.json` 的数据指纹一起解读。该通道缺少严格历史 ST、行业、市值、整手、最低佣金、涨跌停排队、容量部分成交和重试，只能作为权重级方向验证。严格成交结论仍需 Tushare v4 缓存或未来模拟盘。

## 未完成的实证条件

v1.6 不声称达到 15% 年化。晋级前仍需：

1. 本地用同一公开缓存运行四臂，并检查开发、验证、已查看历史保留期是否方向一致；
2. 用严格缓存复核组合上限、停牌/涨跌停、部分成交和成本；
3. 在 2026 年以后未参与改参的数据或模拟盘上做前瞻记录；
4. 用真实券商成交校准平方根冲击系数和上限；校准前不得根据历史收益反向调低成本。
